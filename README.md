# WXT + React

This template should help get you started developing with React in WXT.


Давай начнем разработку с Popup.
У Popup будет 2 основных функционала: аутентификация и добавление новых Bookmark. Мы начнем работу с аутентификацией и потом перейдем к добавлению новых Bookmark. 
У аутентификации есть 2 метода - войти через почту (здесь мы будет отправлять OTP код пользователю на почту) и через OAuth Яндекс.

После установки и открытия Popup пользователь попадает на экран авторизации. На экране авторизации будет возможность войти указав email, OAuth через кнопку войти через Яндекс.
1. Если пользователь ввел email и нажал войти, то он переходит на следующий экран, где можно ввести 6 значный номер. Затем, если код верный, то он авторизаован. Если нет, то выводится ошибка.
2. Если пользователь нажал на кнопку "Войти через Яндекс", тогда открывается вкладка с авторизацией через Яндекс. Этот use case мы реализуем позже.
Сгенерируй эти экраны: аутентификация и валидация OTP кода.


Теперь перейдем к функционалу добавления новых Bookmark через Popup. При добавлении новых Bookmark мы можем выбрать теги или найти их, а также указать в какую collection поместить Bookmark. Также есть кнопка перейти на главный экран приложения.

Еще несколько уточняющих вопросов:
1) Можем ли мы в Popup на стадии добавления новой Bookmark проверить поддерживать ли этот сайт iframe, т.е. можем ли мы потом сделать Live preview.
2) Если ответ на первых вопросов положительно, тогда можем ли мы например сделать скриншот этого сайта  в Desktop / Tablet / Mobile версиях, чтобы потом их выводить в Live Preview. И насколько это бъет по производительности и оптимизации?



Давай начнем реализовывать Library. Сначала сделаем Table View, а затем уже Grid с Live Preview.


Хорошо, следующий шагом мы сделаем возможность редактирования Bookmark. У нас будет открываться Drawer из shadcnui справо. Мы можем менять title, добавлять или удалять tags. А затем перейдем к Grid View. С Collection еще рано работать.
